Detailed Description

Construct volumes on the host.

Currently this requires the full volume attribute exported from SCALE-ORANGE, but it can be reworked in the future to calculate attributes (such as number of intersections) at construction time. We can also implement deduplication of the logic references to reduce memory storage and improve locality. Finally, when we add support for multiple universes, we might need to add a surface ID mapping for the volume input, since the face IDs from one universe won't match the stored global face IDs inside Celeritas-ORANGE.

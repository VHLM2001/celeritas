Detailed Description

Data for surfaces within a single unit.

Surfaces each have a compile-time number of real data needed to define them. (These usually are the nonzero coefficients of the quadric equation.) The two fields in this record point to the collapsed surface types and linearized data for all surfaces in a unit.

The "types" and "data offsets" are both indexed into using the local surface ID. The result of accessing "data offset" is an index into the real_ids array, which then points us to the start address in reals. This marks the beginning of the data used by the surface. Since the surface type tells us the number of real values needed for that surface, we implicitly get a Span of real values with a single indirection.

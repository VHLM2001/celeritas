Detailed Description

Transfer Celeritas sensitive detector hits to Geant4.

This serves a similar purpose to the G4FastSimHitMaker class for generating hit objects. It must be thread-local because the sensitive detectors it stores are thread-local, and additionally Geant4 makes assumptions about object allocations that cause crashes if the HitProcessor is allocated on one thread and destroyed on another.

Call operator:

  • Loop over detector steps
  • Update step attributes based on hit selection for the detector (TODO: selection is global for now)
  • Call the local detector (based on detector ID from map) with the step

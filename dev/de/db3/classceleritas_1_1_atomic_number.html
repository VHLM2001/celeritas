Detailed Description

Type-safe atomic number identifier.

Atomic numbers have constraints that integers do not (zero is not an atomic number, dividing one atomic number by another is meaningless) so this type improves the semantics and safety of Z.

Generally speaking, Z numbers are used during setup and when evaluating expressions that use atomic charge. Z should not be used to "index" data during runtime.

Constructing with a nonpositive Z gives the result a "false" state (get will throw a DebugError).
